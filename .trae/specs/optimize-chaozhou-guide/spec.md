# 潮州旅游攻略全面优化 Spec

## Why

当前 VuePress 潮州旅游攻略项目存在以下问题：
- 多处距离数据标注"⚠️ 待核实"（西湖公园、石板桥、慧如公园及热门打卡表中所有"距英姐美宿"项）
- 图片来源文档仍包含已移除的汕头/南澳岛内容，与主线行程不一致
- 避坑提醒文档编号不连续（缺少"坑5"，从坑4直接跳到坑6）
- 预算汇总文档提及"晓悦楼有免费专属停车位"，但实际住宿为英姐美宿
- frontmatter 不一致（部分文件为空 `---\n---`，部分含 tags）
- 部分文档未加入侧边栏（checklist.md）
- 牌坊数量数据不一致（README 说"22座"，热门打卡说"23座"）
- 门票、营业时间、灯光秀等易变信息需联网最新核实
- 每日行程节奏、餐饮安排、旅行建议仍有优化空间

本次优化旨在通过联网核实 + 内容重组，使攻略在**真实性、时效性、旅行体验、文档质量**四个维度全面提升，同时保留原作者"松弛·深度·有质感"的写作风格。

## What Changes

### 一、信息核实与更新（联网验证）
- **门票价格核实**：己略黄公祠、许驸马府、广济桥、淡浮院、青岚怪臼谷、潮人公园等所有景点门票
- **开放时间核实**：开元寺、广济桥、韩文公祠、己略黄公祠、许驸马府、泰佛殿、淡浮院、青岚怪臼谷
- **闭馆日核实**：确认周一闭馆景点清单（韩文公祠、己略黄公祠、许驸马府等）
- **灯光秀时间核实**：2026年最新灯光秀场次、时间、节假日安排
- **距离数据核实**：英姐美宿 → 各景点的实际驾车/步行距离（高德地图数据）
- **餐厅信息核实**：营业状态、人均价格、营业时间、是否需预订
- **预约方式核实**：韩文公祠"潮玩凤城"小程序、淡浮院网上购票、青岚怪臼谷网上购票
- **潮人公园开园信息核实**：是否如期 2026.1.3 开园、收费项目明细
- **无法确认的信息**：明确标记"⚠️ 需联网验证"，绝不虚构

### 二、行程节奏与餐饮优化
- **核查每日步行距离与交通时间**：避免过度赶路，特别是 Day 2（5景点+餐饮+夜逛）
- **按景点位置合理安排三餐**：补充每餐就近推荐，标注与上一/下一景点的步行距离
- **餐厅推荐补充**：营业时间、人均预算、招牌菜、评价、是否支持预订/大桌
- **错峰建议**：网红店错峰用餐时间建议
- **儿童友好度标注**：3小孩（1.2-1.5m）的餐饮分量、口味适配建议

### 三、旅行建议补充
- **天气详情**：7月潮州气温、降雨、台风概率、紫外线指数
- **着装建议**：按场景细分（古城漫步、青岚徒步、茶馆、寺庙）
- **支付方式**：现金使用场景、移动支付覆盖情况、建议携带现金金额
- **预约方式汇总**：所有需预约的景点/餐厅统一说明
- **避坑提醒扩充**：补充支付、通讯、医疗、紧急联系等实用提醒

### 四、文档质量与 VuePress 兼容性
- **frontmatter 统一**：所有文档添加规范的 tags frontmatter
- **侧边栏补全**：将 checklist.md 加入 guideSideBar.ts
- **内容一致性修复**：
  - 图片来源.md 移除汕头/南澳岛相关条目（与主线行程一致）
  - 预算汇总.md 修正"晓悦楼"为"英姐美宿"
  - 避坑提醒.md 补齐"坑5"或重新编号
  - 牌坊数量统一（22座/23座需核实）
  - 各文档间距离数据一致
- **Markdown 结构优化**：标题层级、表格格式、链接规范、emoji 使用一致性
- **交叉引用链接检查**：确保所有内部链接可达
- **保留原作风格**：仅优化内容质量，不改变"松弛·深度·有质感"的叙事风格

### 五、修改说明文档
- 对所有修改提供"修改说明"（修改前/修改后/修改原因）
- 在每个文档末尾或独立 CHANGELOG 中记录

## ADDED Requirements

### Requirement: 联网信息核实
The system SHALL 通过 WebSearch 核实所有标注"⚠️ 待核实"及易变信息（门票/营业时间/距离/灯光秀），并在文档中以"✅ 已核实（来源：xxx，日期）"或"⚠️ 需联网验证"明确标注。

#### Scenario: 距离数据核实
- **WHEN** 核实英姐美宿到各景点的距离
- **THEN** 通过 WebSearch 查询高德地图数据，更新为实际距离，并移除"⚠️ 待核实"标记

#### Scenario: 无法确认的信息
- **WHEN** 联网搜索后仍无法确认的信息
- **THEN** 明确标记"⚠️ 需联网验证：[核实方法]"，绝不虚构内容

### Requirement: 行程节奏优化
The system SHALL 核查每日步行距离与交通时间，对过度赶路的日程提出优化建议，并按景点位置合理安排三餐。

#### Scenario: 行程过密识别
- **WHEN** 某日景点数量过多或步行距离过长
- **THEN** 在修改说明中标注"建议优化点"，提供节奏调整方案

### Requirement: 修改说明
The system SHALL 对所有修改提供修改说明，包含修改前/修改后/修改原因。

#### Scenario: 修改记录
- **WHEN** 修改任何文档内容
- **THEN** 在独立的修改说明中记录，便于用户审阅

## MODIFIED Requirements

### Requirement: 文档一致性
所有文档 SHALL 保持内容一致：
- 图片来源.md 仅包含主线行程涉及的景点
- 预算汇总.md 的住宿信息与住宿.md 一致
- 避坑提醒.md 编号连续
- 各文档间距离、价格、时间数据一致
- 牌坊数量等事实数据统一

### Requirement: VuePress 兼容性
所有 Markdown 文档 SHALL 符合 VuePress 渲染规范：
- frontmatter 格式统一（含 tags）
- 标题层级规范（H1 仅一处，层级递进）
- 表格格式正确
- 内部链接使用相对路径且可达
- 侧边栏配置完整

## REMOVED Requirements

**Reason**: 主线行程已移除汕头/南澳岛（README 明确说明），但图片来源.md 仍保留相关条目，造成文档不一致
**Migration**: 从图片来源.md 移除汕头小公园、南澳岛灯塔、南澳青澳湾、南澳海鲜等条目；保留的酒店实拍链接中汕头/南澳酒店条目同步移除
